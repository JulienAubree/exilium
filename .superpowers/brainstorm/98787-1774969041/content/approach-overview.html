Page Énergie — 3 Approches

Chaque approche intègre : carte planète, vue flux + vue tableau, knobs de réglage

A

Le flux occupe toute la largeur. Les knobs sont intégrés directement dans le diagramme de flux, à côté de chaque consommateur. Les ressources impactées s'affichent en bas. L'onglet "Tableau" bascule vers une vue liste compacte.

✓ Très visuel et immersif
✓ On voit d'un coup d'œil où va l'énergie
✗ Knobs dans le flux = potentiellement chargé sur mobile

Layout en 2 colonnes. À gauche : bilan énergétique synthétique (produit − consommé = solde). À droite : les bâtiments avec knob + impact sur la ressource. Onglet flux/tableau pour basculer la colonne droite.

✓ Bilan toujours visible pendant les réglages
✓ Chaque knob montre son impact direct
✗ Moins spectaculaire visuellement, plus "utilitaire"

⚡ Hybrid — Bilan visuel + contrôles détachés

│ 🪐 Kepler-442b · Classe M · +15% silicium │

Le flux en haut (lecture seule), les knobs en bas (contrôle). Le diagramme de flux est propre car les contrôles sont séparés. La rangée de knobs en bas forme un "panneau de commande". Le flux s'anime en temps réel quand on tourne un knob.

✓ Flux lisible + contrôles accessibles
✓ Les knobs en rangée = très bon sur mobile (scroll horizontal)
✓ Feedback visuel : le flux réagit aux changements
✗ Plus de scroll vertical (flux + contrôles empilés)

💡 Ma recommandation : C (Hybrid)

La séparation flux/contrôles résout le problème mobile : le flux reste lisible, les knobs sont tous accessibles en bas de page dans une zone dédiée. Et l'animation du flux quand on ajuste un knob donne un feedback immédiat et satisfaisant. C'est le meilleur compromis entre visuel impactant et UX pratique.
