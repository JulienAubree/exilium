Comment ca marche : La file est une liste d'unites individuelles. Les 2 slots piochent chacun la prochaine unite disponible, independamment. Un batch de "5x Recycleur" est decoupe en 5 unites dans la file. Quand un slot finit, il prend la prochaine unite en attente. Resultat : la production est ~2x plus rapide sans changer les temps unitaires.
